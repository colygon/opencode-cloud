# opencode.cloud

**Hosted AI coding agent with branching execution and real-time collaboration**

opencode.cloud is a serverless platform that brings [opencode](https://github.com/anomalyco/opencode) — the open-source AI coding agent — to the web. Every user gets an isolated sandbox running opencode, accessible from any browser with zero local setup.

## What Makes It Special

### 🌳 Branching Execution

Unlike other hosted coding environments, opencode.cloud lets you:
- **Checkpoint** the AI's progress at any point
- **Fork** into parallel exploration branches
- **Try multiple approaches** simultaneously from the same starting point
- **Rollback** instantly if something goes wrong
- **Compare outcomes** and keep the best result

This enables **Monte Carlo Tree Search over solution strategies** — something no other platform offers.

### 👥 Real-Time Collaboration

- Share a session URL with your team
- Multiple users edit files simultaneously with conflict-free CRDT sync
- See collaborators' cursor positions in real-time
- Chat with the AI together

### ⚡ Zero Setup

No installation, no API keys to manage, no environment configuration. Click a button and start coding with AI.

## Key Features

| Feature | Description |
|---------|-------------|
| **Isolated Sandboxes** | Each session runs in its own MicroVM with full Linux environment |
| **Immutable Snapshots** | Git-like checkpoints of execution state for branching and rollback |
| **Multi-User Editing** | Yjs CRDT for conflict-free concurrent editing |
| **Session Persistence** | State survives disconnects and server restarts |
| **Real Tools** | Full access to git, npm, shell, and everything opencode provides |
| **Enterprise Auth** | JWT + API key authentication with role-based access |

## Why Contree?

opencode.cloud runs on **Contree** — a managed sandbox platform with unique capabilities:

- **Real Linux environments** — No container limitations, everything opencode needs works natively
- **Immutable snapshots** — Git-like branching with instant rollback
- **VM-level isolation** — Stronger security than containers
- **No infrastructure to manage** — We focus on features, not DevOps
